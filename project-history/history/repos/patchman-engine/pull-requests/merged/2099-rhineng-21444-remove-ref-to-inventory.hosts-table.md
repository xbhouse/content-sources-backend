## Summary by Sourcery

Migrate usage of inventory metadata from the external inventory.hosts table into the local system_inventory table, including workspaces/groups, workloads, tags, OS and timestamps, and update schema, data fixtures, and tests to reflect the new source of truth.

New Features:
- Store inventory workspaces/groups as JSONB on system_inventory and backfill them from existing inventory host groups via a database migration.

Bug Fixes:
- Fix workload, group, and SAP SID filtering to operate on system_inventory fields rather than JSON system_profile data on inventory.hosts.
- Correct handling of system workspaces by storing full group objects instead of only group IDs and ensuring filtering and exports use the new representation.
- Resolve inconsistencies in package cache expectations, package availability, and repository-based inventory lookups that caused previous tests to rely on outdated data assumptions.

Enhancements:
- Change query helpers, controllers, and background tasks to join against system_inventory instead of inventory.hosts for tags, groups, OS attributes, timestamps, and workload-related filters.
- Model subscription manager ownership as a UUID-based OwnerID/SystemProfile field and persist it directly on system_inventory.
- Align package cache and VMaaS-related logic and tests with the updated package data, including latest kernel package selection and package counts.

Tests:
- Refresh SQL test data and Go tests across controllers, evaluators, tasks, and listeners to validate behavior against system_inventory-based metadata, new workspaces format, and updated package/erratum expectations.

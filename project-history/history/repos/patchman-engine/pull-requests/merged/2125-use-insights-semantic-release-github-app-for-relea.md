Hey - I've found 1 issue, and left some high level feedback:

- The `Generate token` step appears to be mis-indented in the YAML (the `- name`, `id`, and `uses` lines should be aligned at the same level as the other steps) which will prevent the workflow from loading correctly.
- When switching to `actions/create-github-app-token@v1`, double-check that the GitHub App installation has `contents: write` permission on this repo so the checkout and subsequent push operations can succeed with the new token.

**issue (bug_risk):** Fix indentation for the new step to ensure valid GitHub Actions YAML

In the new step, `id`, `uses`, and `with` are indented too far compared to `name`. All keys for a single step must align under the `-` marker. It should look like:

```yaml
    - name: Generate token
      id: app-token
      uses: actions/create-github-app-token@v1
      with:
        app-id: ${{ secrets.SEMANTIC_RELEASE_APP_ID }}
        private-key: ${{ secrets.SEMANTIC_RELEASE_PK }}
```

The current indentation will cause the workflow to be parsed incorrectly.

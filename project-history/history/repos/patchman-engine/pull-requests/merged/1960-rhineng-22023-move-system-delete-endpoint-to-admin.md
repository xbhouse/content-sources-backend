## Summary by Sourcery

Move the system deletion functionality from the user-facing systems API to the admin API and centralize controller error-handling helpers.

New Features:
- Expose an admin-only DELETE /system/{inventory_id} endpoint to delete systems by inventory ID and account ID.

Enhancements:
- Introduce shared controller utility helpers for logging and HTTP error responses in the base utils package and update manager controllers to use them.
- Add HTTP request/route test helper functions for controller tests in the manager package.
- Update API routing and OpenAPI documentation to reflect the new admin system deletion endpoint and removal of the user-facing deletion route.

Tests:
- Add and update controller tests to cover the new admin system delete endpoint, including success, wrong account, not found, and invalid inventory ID cases.

#### Sequence diagram for the new admin system delete endpoint

```mermaid
sequenceDiagram
    actor AdminUser
    participant GinAdminRouter
    participant AdminController as AdminSystemDeleteHandler
    participant DB as Database
    participant DBFunc as DBFunction_delete_system
    participant Utils as BaseUtilsController

    AdminUser->>GinAdminRouter: DELETE /api/patch/v1/admin/system/{inventory_id}?rh_account_id=...
    GinAdminRouter->>AdminController: SystemDeleteHandler(c)

    AdminController->>AdminController: Parse rh_account_id query param
    alt Invalid_rh_account_id
        AdminController->>Utils: LogAndRespBadRequest(c, err, "rh_account_id query param not found")
        Utils-->>AdminUser: 400 Bad Request
        AdminController->>AdminController: return
    end

    AdminController->>AdminController: Read inventory_id path param
    alt Missing_inventory_id
        AdminController-->>AdminUser: 400 Bad Request (inventory_id param not found)
        AdminController->>AdminController: return
    end

    AdminController->>AdminController: Validate inventory_id UUID
    alt Invalid_inventory_id
        AdminController->>Utils: LogAndRespBadRequest(c, err, "incorrect inventory_id format")
        Utils-->>AdminUser: 400 Bad Request
        AdminController->>AdminController: return
    end

    AdminController->>DB: Begin transaction
    DB-->>AdminController: tx

    AdminController->>DB: SELECT inventory_id FROM system_platform FOR UPDATE
    alt Query_error
        AdminController->>Utils: LogAndRespError(c, err, "could not query database for system")
        Utils-->>AdminUser: 500 Internal Server Error
        DB-->>AdminController: Rollback
        AdminController->>AdminController: return
    end

    alt No_matching_system
        AdminController->>Utils: LogAndRespNotFound(c, err, "system not found")
        Utils-->>AdminUser: 404 Not Found
        DB-->>AdminController: Rollback
        AdminController->>AdminController: return
    end

    AdminController->>DBFunc: select deleted_inventory_id from delete_system(inventory_id)
    alt Delete_error
        AdminController->>Utils: LogAndRespError(c, err, "Could not delete system")
        Utils-->>AdminUser: 500 Internal Server Error
        DB-->>AdminController: Rollback
        AdminController->>AdminController: return
    end

    AdminController->>DB: Commit transaction
    alt Commit_error
        AdminController->>Utils: LogAndRespError(c, err, "Could not delete system")
        Utils-->>AdminUser: 500 Internal Server Error
        AdminController->>AdminController: return
    end

    alt Rows_deleted
        AdminController-->>AdminUser: 200 OK
    else No_rows_deleted
        AdminController->>Utils: LogAndRespNotFound(c, err, "system not found")
        Utils-->>AdminUser: 404 Not Found
    end
```

Hey there - I've reviewed your changes - here's some feedback:

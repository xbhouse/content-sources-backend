## Summary by Sourcery

Improve the database schema migration workflow by including the manager role in connection-blocking logic and tightening database connection settings.

Bug Fixes:
- Ensure the manager database user is included in migration-time session checks and login blocking/unblocking to avoid interfering active sessions during schema updates.

Enhancements:
- Log messages for active session checks now explicitly mention the manager user alongside existing roles.
- Configure PostgreSQL connection URLs to use explicit TCP keepalive parameters to make database connections more robust during migrations.

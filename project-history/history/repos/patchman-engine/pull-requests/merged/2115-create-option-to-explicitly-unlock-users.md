Hey - I've found 1 issue, and left some high level feedback:

- The new `unlockUsers` function name conflicts with the `unlockUsers` config boolean in `config.go`, which will prevent compilation and makes the intent in `UpdateDB` (`if unlockUsers { unlockUsers(db) }`) ambiguous—rename either the function or the config variable to keep types and purposes distinct.
- In `UpdateDB`, consider guarding `unlockUsers(db)` with the same existence/creation assumptions as for user-related operations elsewhere (e.g., `updateUsers`/schema creation), or handling missing users more gracefully, since calling `execOrPanic` on non-existent roles will abort the whole admin operation.

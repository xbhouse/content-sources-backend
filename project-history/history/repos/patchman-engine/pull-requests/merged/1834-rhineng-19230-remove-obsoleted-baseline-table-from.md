## Summary by Sourcery

Remove the obsolete baseline feature by dropping its schema objects, deleting associated code paths and tests, and refactor recalc logic to focus solely on template-based evaluation.

Enhancements:
- Drop the baseline table and its partitions from the schema, and remove baseline_id and baseline_uptodate columns and constraints from system_platform.
- Delete the Baseline GORM model and all related database helpers, migrations, and secure coding test utilities.
- Refactor the Kafka recalc pipeline: rename runBaselineRecalcLoop to runRecalcLoop, replace EvaluateBaselineSystems with RecalcSystems, and eliminate baseline evaluation config flags.
- Update API controllers to always return default baseline attributes, remove baseline-based filtering and sorting, and adjust common attributes to use empty or zero values.
- Clean up tests and test data: remove baseline SQL inserts, delete baseline-specific tests and assertions, and update metrics_db test expectations to reflect the removed table.

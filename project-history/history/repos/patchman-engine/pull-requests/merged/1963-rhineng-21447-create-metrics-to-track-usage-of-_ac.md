## Reviewer's Guide

Adds Prometheus metrics to track cache hits and misses for advisory_account_data and package_account_data, and wires them into the advisories and packages query paths.

#### Sequence diagram for advisory_account_data cache hit/miss metrics

```mermaid
sequenceDiagram
    actor Client
    participant GinRouter
    participant AdvisoriesController
    participant Middlewares
    participant DB

    Client->>GinRouter: HTTP GET /advisories
    GinRouter->>AdvisoriesController: advisoriesCommon(c)
    AdvisoriesController->>DB: buildBaseQuery(db, filters, account)
    DB-->>AdvisoriesController: *gorm.DB

    alt DisableCachedCounts or HasInventoryFilter or groups_not_empty
        AdvisoriesController->>Middlewares: AdvisoryAccountDataHitsCnt.WithLabelValues(hit).Inc()
        AdvisoriesController->>DB: buildQueryAdvisoriesTagged(db, filters, account, groups)
    else use advisory_account_data cache
        AdvisoriesController->>Middlewares: AdvisoryAccountDataHitsCnt.WithLabelValues(miss).Inc()
        AdvisoriesController->>DB: buildQueryAdvisories(db, account)
    end

    DB-->>AdvisoriesController: advisories result set
    AdvisoriesController-->>GinRouter: HTTP response payload
    GinRouter-->>Client: 200 OK
```

#### Sequence diagram for package_account_data cache hit/miss metrics

```mermaid
sequenceDiagram
    actor Client
    participant GinRouter
    participant PackagesController
    participant Middlewares
    participant DB

    Client->>GinRouter: HTTP GET /packages
    GinRouter->>PackagesController: packagesQuery(db, filters, acc, groups, useCache)

    alt useCache is true
        PackagesController->>Middlewares: PackageAccountDataHitsCnt.WithLabelValues(hit).Inc()
        PackagesController->>DB: SELECT FROM package_account_data JOIN package_name WHERE rh_account_id = acc
        DB-->>PackagesController: cached package rows
    else useCache is false
        PackagesController->>Middlewares: PackageAccountDataHitsCnt.WithLabelValues(miss).Inc()
        PackagesController->>DB: Systems(db, acc, groups) query with packages_installed > 0
        DB-->>PackagesController: live package rows
    end

    PackagesController-->>GinRouter: HTTP response payload
    GinRouter-->>Client: 200 OK
```

Hey there - I've reviewed your changes - here's some feedback:

- The new AdvisoryAccountDataHitsCnt and PackageAccountDataHitsCnt metrics don’t appear to be registered in the Prometheus() middleware initializer, so they won’t be exposed unless you add them to the MustRegister call.
- Using a very generic label name like `type` for the hit/miss dimension makes the metric harder to interpret; consider a more descriptive label (e.g. `cache_status`) to improve clarity and avoid collisions with other metrics.
- To avoid stringly-typed label values and potential typos, consider defining constants for the `hit` and `miss` label values used with the new counter metrics.

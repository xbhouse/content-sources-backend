## Summary by Sourcery

Reimplement repo-based recalculation by consolidating repo and package retrieval, simplifying inventory ID queries, and adjusting sync and cache refresh flow

Enhancements:
- Unify VMAAS repository and package retrieval into a single getVmaasRepos function with dedicated getUpdatedRepos and getUpdatedReposWithPackages helpers
- Simplify getRepoBasedInventoryIDs to perform a single SQL join filtering on provided repos and packages
- Refactor syncRepos to use streamlined repo list for non-third-party repositories
- Relocate cache refresh to after timestamp updates in SyncData
- Update platform.vmaas.go reposHandler sample payload to include updated_package_names

Tests:
- Update and rename unit tests to match the new function signatures and behavior for getting repos, packages, and inventory IDs

// before:
func getUpdatedReposWithPackages(syncStart time.Time, modifiedSince *string) ([]string, []string, *time.Time, error) {
  var affectedRepos []string
  var affectedPackages []string

  repoMap, lastChange, err := getVmaasRepos(syncStart, modifiedSince, true)
  if err != nil {
    return nil, nil, nil, err
  }

  included := make(map[string]bool)
  for repoName, packageList := range repoMap {
    hasPackage := false
    for _, packageName := range packageList {
      if !included[packageName] {
        included[packageName] = true
        affectedPackages = append(affectedPackages, packageName)
      }
      hasPackage = true
    }
    if hasPackage {
      affectedRepos = append(affectedRepos, repoName)
    }
  }

  return affectedRepos, affectedPackages, lastChange, nil
}

// after:
func getUpdatedReposWithPackages(syncStart time.Time, modifiedSince *string) ([]string, []string, *time.Time, error) {

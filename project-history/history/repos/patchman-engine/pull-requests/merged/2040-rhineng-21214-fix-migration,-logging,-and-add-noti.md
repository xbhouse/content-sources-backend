```mermaid
sequenceDiagram

    participant migrateUp as MigrateUp
    participant migrateAction as migrateAction

        migrateUp->>app: panic(err)
    end

    app->>migrateAction: migrateAction(conn, sourceURL)
    migrateAction->>utils: LogInfo("schema", expectedSchema, "DB migration in progress, waiting for schema")
    migrateAction->>db: dbSchemaVersion(conn, sourceURL)
    db-->>migrateAction: dbSchema or error
    alt dbSchemaVersion error and dirty
        migrateAction-->>app: MIGRATE
    else dbSchemaVersion error other
        migrateAction->>utils: LogError("err", err.Error(), "error getting current DB version")
        migrateAction-->>app: BLOCK
    end

    migrateAction->>db: latestSchemaMigrationFileVersion(sourceURL)
    db-->>migrateAction: migrationSchema or error
    alt latestSchemaMigrationFileVersion error
        migrateAction->>utils: LogError("err", err.Error(), "failed to load latestSchemaMigrationFileVersion")
        migrateAction-->>app: BLOCK
    else schema checks
        alt missing migration files
            migrateAction->>utils: LogError("schema", migrationSchema, "missing migration files for schema")
            migrateAction-->>app: BLOCK
        else deployment intentionally blocked
            migrateAction->>utils: LogWarn("deployment blocked, enable migrations to proceed")
            migrateAction-->>app: BLOCK
        else db upgraded
            migrateAction->>utils: LogInfo("DB is upgraded")
            migrateAction-->>app: CONTINUE
        else need migration
            migrateAction->>utils: LogInfo("current version:", dbSchema, "expected:", expectedSchema, "migrateAction")
            migrateAction-->>app: MIGRATE
        end
    end

    app->>db: NewConn(databaseURL) notice handler
    db-->>app: pq.Error notice
    app->>utils: LogInfo("Notice: " + e.Error())
```

<issue_to_address>
**issue (bug_risk):** Top-level RAISE NOTICE statements may not be valid outside PL/pgSQL blocks.

In PostgreSQL, `RAISE NOTICE` is only valid inside PL/pgSQL (e.g., inside a function or `DO $$ BEGIN ... END $$;`). At the top level of a migration script it will cause a syntax error. If you need these messages, wrap them in a `DO` block (e.g., `DO $$ BEGIN RAISE NOTICE 'Created system_inventory'; END $$;`) or rely on the migration system’s existing logging instead.

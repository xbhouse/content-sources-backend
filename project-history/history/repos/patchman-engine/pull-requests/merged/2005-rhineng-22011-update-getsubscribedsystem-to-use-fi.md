## Reviewer's Guide

Updates getSubscribedSystem to use GORM's First instead of Find while explicitly treating ErrRecordNotFound as a non-fatal case, improving query semantics and error handling for missing subscribed systems.

#### Sequence diagram for updated getSubscribedSystem database lookup and error handling

```mermaid
sequenceDiagram
    participant GinContext as GinContext
    participant Controller as getSubscribedSystem
    participant DB as GormDB
    participant Utils as Utils

    GinContext->>Controller: getSubscribedSystem(c, tx)
    activate Controller

    Controller->>DB: Table("inventory.hosts ih")
    DB-->>Controller: *gorm.DB

    Controller->>DB: Joins("JOIN rh_account acc on ih.org_id = acc.org_id")
    DB-->>Controller: *gorm.DB

    Controller->>DB: Where("ih.system_profile->>'owner_id' = ? AND acc.id = ?", systemCn, account)
    DB-->>Controller: *gorm.DB

    Controller->>DB: First(&inventoryID)
    DB-->>Controller: error err

    alt database error and not ErrRecordNotFound
        Controller->>Utils: LogAndRespError(c, err, "database error")
        Utils-->>Controller: void
        Controller-->>GinContext: returns (0, "", err)
    else no error or ErrRecordNotFound
        Controller-->>GinContext: returns (inventoryID, systemCn, nil or ErrRecordNotFound)
    end

    deactivate Controller
```

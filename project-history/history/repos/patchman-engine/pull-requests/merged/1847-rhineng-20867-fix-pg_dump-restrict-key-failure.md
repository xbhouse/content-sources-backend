## Summary by Sourcery

Fix the pg_dump restrict-key issue by centralizing the dump logic in a helper and updating tests to use the new functionality.

Bug Fixes:
- Include the `--restrict-key testupgradekey` flag in the pg_dump invocation to correctly apply the restrict key.

Enhancements:
- Consolidate the pg_dump command construction and execution into a single `dumpSchema()` helper function.
- Remove the `setCmdAuth` helper and redundant pg_dump setup in tests.

Tests:
- Refactor schema compatibility tests to use the new `dumpSchema()` helper instead of manual pg_dump commands.

Hey there - I've reviewed your changes - here's some feedback:

- Parameterize the restrict key instead of hardcoding "testupgradekey" to allow flexibility and reuse across environments.
- Use CombinedOutput or explicitly capture stderr in dumpSchema so that pg_dump errors are surfaced in test failure messages for easier debugging.
- Consider adding a version check or fallback behavior for pg_dump in case --restrict-key isn’t supported on older installations to avoid test breakage.

Hey there - I've reviewed your changes - here's some feedback:

- In initKesselFromEnv you’re defaulting KESSEL_INSECURE to true instead of falling back to the existing CoreCfg.KesselInsecure—consider using CoreCfg.KesselInsecure as the default for consistency.
- In setupClient when KesselInsecure is false you still aren’t loading any TLS credentials—make sure to configure root CAs or certificate credentials rather than just toggling the insecure flag.
- The ClowdApp manifest hardcodes a secret named "kessel-service-client" for auth; consider parameterizing the secret name or adding a note about creating that secret to avoid deployment surprises.

## Description

## Summary

Improves query by:

* Adding Partial indexes specifically for ready tasks filtering
* Adding Composite indexes for priority-based ordering
* Dependency indexes to optimize the expensive NOT IN subquery
* Optimized view using EXISTS instead of NOT IN

After merge, i'll monitor stage & production for a drop in cpu load as well as re-run the explain plan after its merged to stage.

Indexes and query generated by Cursor

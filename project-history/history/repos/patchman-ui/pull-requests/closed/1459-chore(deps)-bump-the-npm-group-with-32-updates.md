<h2>Changes</h2>
<h3>React Native Peer Dependency Removed</h3>
<p>We've always had an awkward peer dependency on <em>both</em> ReactDOM and React Native, because of the need to import the <code>unstable_batchedUpdates</code> API directly from each reconciler. That's part of what led to the sequence of 9.x patch releases to deal with RN compat.</p>
<p>As of 9.0.3, we dropped the batching imports completely, since React 18 now batches by default.  That means we didn't even have any remaining imports from <code>react-native</code>.</p>
<p>Meanwhile, React 18.3 just came out, but so did React Native 0.74. RN 0.74 still requires React 18.2.</p>
<p>This caused NPM users to have installation failures when trying to use React-Redux:</p>
<ul>
<li>React-Redux has a peer dep on RN</li>
<li>RN has a peer dep on React 18.2</li>
<li>But the <em>latest</em> React, 18.3 would get installed in the app</li>
<li>NPM errors with a peer dep mismatch</li>
</ul>
<p>We no longer need to list RN as a peer dep, and dropping that also fixes the NPM installation issues as well.</p>
